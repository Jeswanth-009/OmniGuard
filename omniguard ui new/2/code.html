The Request Lifecycle

01

Edge Interception

Inbound TCP packets are terminated at the Nginx edge. Initial SSL/TLS handshakes are handled via optimized BoringSSL.

02

Context Enrichment

Metadata including IP geolocation, ASN reputation, and device fingerprinting is attached to the request header.

03

FastAPI Validation

Python-based high-concurrency workers parse the payload. Pydantic models ensure schema integrity at the door.

04

Cache Lookaside

Redis clusters are queried for existing session tokens and rate-limit counters, avoiding expensive DB roundtrips.

05

Threat Analysis

Vectorized signature matching runs against known exploit patterns (SQLi, XSS, RCE) in parallel execution threads.

06

Proxy Dispatch

Upon clearance, the sanitized request is proxied to the upstream origin with an injected Omni-Guard-Signed header.
